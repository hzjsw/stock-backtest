## 测试步骤

### 步骤 1: 重启后端服务器

**重要**: 由于端口占用，需要手动重启后端服务器。

**方法 1: 使用任务管理器**
1. 按 `Ctrl+Shift+Esc` 打开任务管理器
2. 找到所有 `node.exe` 进程
3. 右键 → 结束任务

**方法 2: 使用命令**
```powershell
# 在 PowerShell 中执行
Get-Process node | Stop-Process -Force
```

**方法 3: 使用 netstat 找到并杀掉进程**
```powershell
# 找到占用 3001 端口的进程 ID
netstat -ano | findstr :3001

# 杀掉进程 (替换 PID 为实际数字)
taskkill /F /PID <PID>
```

**启动后端服务器**:
```bash
cd "d:\AI IDE PRO\stock-backtest"
npm run server
```

### 步骤 3: 测试在线数据获取

1. 打开浏览器访问：`http://localhost:5175`

2. 在左侧面板：
   - 点击 **"在线获取"** 按钮
   - 数据源选择：**网易财经 (推荐)**
   - 股票代码：输入 `600000, 600036`
   - 开始日期：`20230101`
   - 结束日期：`20231231`
   - ✅ 勾选 **"自动保存到 data/ 目录"**
   - 点击 **"获取数据"** 按钮

3. 查看提示消息：
   ```
   ✓ 成功获取 2 只股票的数据
   ✓ 数据已保存到 data/ 目录
   ```

4. 检查文件是否生成：
   - 打开 `d:\AI IDE PRO\stock-backtest\data\` 目录
   - 应该看到 `600000.csv` 和 `600036.csv` 文件

### 步骤 4: 测试回测功能

1. 在左侧面板选择策略，例如 **"双均线交叉"**

2. 调整策略参数（可选）：
   - 短周期：5
   - 长周期：20

3. 点击 **"运行回测"** 按钮

4. 查看结果：
   - 右侧应该显示绩效指标
   - 图表应该显示权益曲线
   - 交易记录表格应该有数据

5. **验证是否使用了真实数据**：
   - 查看浏览器控制台（F12）
   - 应该看到日志输出：
     ```
     发送回测请求：{ dataSource: { type: 'online', symbols: ['600000', '600036'], ... } }
     ```
   - 后端服务器控制台应该显示：
     ```
     ✓ 加载 600000 (6268 条数据)
     ✓ 加载 600036 (5740 条数据)
     ```

## 故障排查

### 问题 1: 点击"获取数据"后没有反应

**检查**:
1. 后端服务器是否运行在 `http://localhost:3001`
2. 浏览器控制台是否有错误（F12）
3. 网络请求是否成功

**解决**:
- 重启后端服务器
- 清除浏览器缓存

### 问题 2: data/ 目录没有生成文件

**检查**:
1. 后端服务器控制台日志
2. 请求是否包含 `autoSave: true`

**解决**:
- 确保勾选了"自动保存到 data/ 目录"
- 检查后端代码是否有 `saveToCsv` 函数

### 问题 3: 回测仍使用模拟数据

**检查**:
1. 回测请求中的 `dataSource.type` 是否为 `'online'`
2. 后端是否从 data/ 目录加载了 CSV 文件

**解决**:
- 确保先获取数据并保存到 data/ 目录
- 检查后端服务器日志，看是否加载了 CSV 文件

### 问题 4: 端口 3001 被占用

**解决**:
```powershell
# 找到占用端口的进程
netstat -ano | findstr :3001

# 杀掉进程（替换 <PID> 为实际数字）
taskkill /F /PID <PID>

# 或者使用 PowerShell
Get-Process | Where-Object {$_.Id -eq <PID>} | Stop-Process -Force
```
