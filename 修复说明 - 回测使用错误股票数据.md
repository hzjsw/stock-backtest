# 修复说明：回测使用错误股票数据的问题

## 问题描述

用户反馈：
- ✅ 数据获取正常
- ✅ 保存的文件正常
- ❌ 运行回测后股票始终是 000001
- ❌ 股价也不对

## 根本原因

在 `src/server/index.ts` 的 `handleBacktest` 函数中，即使已经加载了真实的股票数据（如 600000），代码仍然会检查 `if (!data.has('000001'))`，如果不包含 000001 就会添加模拟数据。

**错误的代码逻辑**：
```typescript
// ❌ 问题代码
case 'ma-crossover':
  strategy = createMACrossover(p.shortPeriod || 5, p.longPeriod || 20);
  if (!data.has('000001')) data.set('000001', generateStockData('000001'));
  break;
```

**问题分析**：
1. 用户获取了 600000 的真实数据并保存到 `data/600000.csv`
2. 回测时，`dataSource.type === 'online'`，成功加载了 600000 的数据
3. 但是进入 `switch (body.strategy)` 后，代码检查 `if (!data.has('000001'))`
4. 因为 data 中有 600000 但没有 000001，条件为 true
5. 于是添加了 000001 的模拟数据
6. 回测引擎同时有 600000（真实）和 000001（模拟）的数据
7. 策略可能使用了 000001 的数据，导致股价不对

## 修复方案

**修复后的代码逻辑**：
```typescript
// ✅ 修复代码
case 'ma-crossover':
  strategy = createMACrossover(p.shortPeriod || 5, p.longPeriod || 20);
  // 只有在没有任何数据时才使用模拟数据
  if (data.size === 0) {
    console.log('  ⚠ 未加载数据，使用模拟数据 000001');
    data.set('000001', generateStockData('000001'));
  }
  break;
```

**修复要点**：
1. 检查 `data.size === 0` 而不是 `!data.has('000001')`
2. 只有当没有任何数据时才使用模拟数据
3. 如果已有真实数据，直接使用
4. 添加日志输出，显示使用的数据

## 常见问题

### Q: 为什么重启后端服务器很重要？

A: 因为修改的是 TypeScript 源代码，需要重新编译和运行。不重启的话，运行的是旧版本的代码。

A:

3. 检查 `data/600000.csv` 文件是否存在

### Q: 如何确认使用了真实数据？

A: 
1. 查看后端日志中的"回测使用数据"
2. 检查交易记录中的股价（600000 约 10 元）
3. 对比 `data/600000.csv` 中的收盘价

## 技术细节

### 修复逻辑

```typescript
// 之前的逻辑（错误）
if (!data.has('000001')) {
  // 只要没有 000001，就添加模拟数据
  // 问题：即使有 600000 的真实数据，也会添加 000001
  data.set('000001', generateStockData('000001'));
}

// 修复后的逻辑（正确）
if (data.size === 0) {
  // 只有没有任何数据时，才使用模拟数据
  // 如果已有真实数据（无论哪个股票），都直接使用
  data.set('000001', generateStockData('000001'));
}
```

### 数据流程

1. **获取数据** → `data/600000.csv`
2. **回测请求** → `dataSource.type = 'online'`
3. **加载数据** → 从 CSV 文件读取 600000 的数据
4. **策略初始化** → 检查 `data.size`
5. **使用数据** → 直接使用已有的 600000 数据
6. **回测执行** → 基于 600000 的真实价格
